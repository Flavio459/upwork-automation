# Template - Pós-Venda e Expansão

> [!IMPORTANT]
> Use este playbook depois da entrega aceita, durante hypercare ou após o handoff final.
> Se esta for uma conversa nova, leia primeiro [[docs/00_Governance/Chat_Bootstrap|Chat Bootstrap]] e [[docs/00_Governance/Reference_Map|Mapa de Referências]].

## Objetivo

Transformar a entrega em ativo comercial reutilizável e capturar a próxima oportunidade antes que o ciclo esfrie.

## Quando Usar

- a entrega principal já foi aceita
- existe sinal explícito de confiança ou continuidade
- o cliente pediu ajuste, suporte ou nova frente
- há chance real de `testimonial`, `case`, `upsell` ou `referral`

## Sinais De Expansão

- pediu manutenção adicional
- perguntou sobre próxima fase
- quer levar o padrão para outra área ou equipe
- comentou que o resultado foi útil para vender internamente
- pediu material para compartilhar com outro decisor

## Ativos Pós-Entrega

- [ ] testimonial solicitado
- [ ] case capturado
- [ ] upsell identificado
- [ ] retainer avaliado
- [ ] referral solicitado

## Sequência Operacional

1. Registrar o estado final da entrega.
2. Identificar qual ativo pós-venda faz mais sentido.
3. Definir a próxima oportunidade de contrato.
4. Localizar qualquer comunicação externa para a língua do cliente.
5. Registrar a ação de follow-up.
6. Encerrar com uma decisão clara: `Expansion` ou `Closed`.

## Gate Antes De Expandir

- entrega principal aceita
- escopo atual encerrado ou em hypercare controlado
- pedido do cliente validado por escrito
- preço da expansão calculado
- comunicação externa revisada se não for `pt-BR`

## Saída Obrigatória

Este playbook só termina quando houver:

- um ativo de expansão escolhido
- a próxima ação registrada
- o status atualizado para `Expansion` ou `Closed`
