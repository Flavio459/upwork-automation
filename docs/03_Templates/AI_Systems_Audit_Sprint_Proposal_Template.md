# AI Systems Audit Sprint - Proposal Template

> [!IMPORTANT]
> Este documento é um artefato de suporte para proposta. Ele acompanha o fluxo operacional, mas nao o substitui.
> O fluxo canônico continua sendo `Opportunity_Intake_and_Fit` -> `Feasibility_and_Capability_Assessment` -> `Costing_Pricing_and_Timeline` -> `Proposal_and_Localization_Pack`.
> Quando existir workspace operacional por lead, use `reports/operator-workspaces/<lead_id>/` como base de entrada.

## Operating Rule

- write the master proposal first in `pt-BR`
- adapt the client-facing version to `client_language`
- review tone, technical precision, and CTA before send
- validate the localized version with `Localization_Review_Checklist.md`

## Cover Letter

```text
Hi [Client Name],

I reviewed your project and the main issue is not a missing tool. It is the lack of a clear AI architecture that connects the work you already do with measurable operational gain.

I help teams turn manual workflows into productized AI systems by running a focused audit, identifying the highest-value opportunities, and turning them into an implementation roadmap.

If you want, I can start with a short discovery sprint and show you exactly where AI can reduce effort, risk, or turnaround time.

---
```

## Scope

```text
## AI SYSTEMS AUDIT SPRINT

### Phase 1: Discovery
- Review the current workflow
- Identify manual steps and bottlenecks
- Clarify goals, constraints, and success criteria

### Phase 2: Audit
- Score the workflow by impact, effort, and risk
- Identify where AI is feasible end-to-end
- Separate high-value opportunities from low-value noise

### Phase 3: Architecture
- Recommend the best workflow architecture
- Define tools, integrations, and data flow
- Show what should be built now vs later

### Phase 4: Roadmap
- Prioritize implementation phases
- Estimate effort and investment
- Prepare the handoff for execution

---
```
