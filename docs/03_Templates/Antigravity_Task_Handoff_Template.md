# Review Contract

Responda de forma barata de auditar. Sua resposta final deve incluir, nesta ordem:

1. `Summary`
- 3 a 6 linhas com o que foi feito.

2. `Files Changed`
- Lista simples de arquivos alterados.

3. `Validation Run`
- Comandos executados.
- Resultado de cada comando.
- Se algo nao foi executado, diga explicitamente.

4. `Key Diffs`
- Mostre apenas 2 a 5 trechos curtos que representem as mudancas centrais.
- Nao despeje arquivos inteiros.

5. `Residual Risks`
- Riscos, limitacoes ou pendencias restantes.

6. `Preserved By Contract`
- Liste tudo que foi mantido de proposito por contrato externo, compatibilidade ou limite de escopo.

# Output Format

- Resumo curto do que foi feito.
- Lista de arquivos alterados.
- Riscos ou pendencias restantes.
